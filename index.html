Metrology Data Analyzer — Version 1.6

May 2026 release notes

UI / UX

  • Home & toolbar — Home button returns to the start page; toolbar grouped next to the app name (open, save, save as, PDF setup, PowerPoint export, theme toggle).
  • Light mode default — App opens in light mode with a smooth cross-fade theme transition; chart backgrounds update in sync with the UI.
  • Chart polish — Light-mode axis lines use soft slate gray instead of harsh black borders.
  • Choose File prompt — Data file picker button pulses orange until an Excel file is loaded.
  • Steel adjustment sync — Standard Analysis and Critical Six-Pack share the same per-dimension steel offset; changing one updates the other.
  • SPC panels — Run verdict names best/worst run by stability; Cavity Stability explains usage %; Tooling / Centering uses a distinct purple accent aligned with other cards.
  • Changelog modal — What's New opens a full release-notes panel organized by category.

Math & Calculations

  • Normality tests — Anderson–Darling (default), Ryan–Joiner, and Kolmogorov–Smirnov available on the Six-Pack tab. Side-by-side comparison shows pass/fail at α = 0.05 and highlights when a non-AD test passes but AD fails.
  • Capability methods — When data fail normality, choose Parametric (with warning), Percentile (non-parametric Pp/Ppk), Box-Cox, Johnson, or Taylor high-capability waiver (STAT-18). Methods are gated when not applicable (e.g., mixed cavities).
  • Transformations — Box-Cox and Johnson fits run automatically for non-normal data; transformed Anderson–Darling P-values (≥ 0.10 Minitab IDI rule) determine whether a transform is viable. Capability stats and histogram update for the selected method.
  • Normality review panel — Consolidated AD vs selected-test readout with recommended capability path and short triage note on the stats card.
  • SPC stability metrics — Run-level usage averages, cavity usage = |value − nominal| ÷ (tolerance ÷ 2) × 100, and risk drivers (Process Variation, Tooling Centering, Instability, Marginal).

Minitab Workflow Suggestions

  • Path Forward card — After Six-Pack analysis, a Minitab-style guide recommends the exact menu path (Normal vs Nonnormal Capability Analysis).
  • Transformation guidance — Suggests None, Box-Cox, or Johnson based on transformed AD results and explains why raw data failed.
  • Distribution context — Notes Johnson system family (Sb / Sl / Su) or post–Box-Cox evaluation steps so you can replicate the study in Minitab IDI.
  • One-click apply — “Apply recommended Box-Cox” shortcut when Box-Cox is the top viable transform.

Six-Pack & Cavity Analysis

  • Cavity Triage panel — Side-by-side mini box plots to compare cavities or runs without leaving the Critical Six-Pack tab.
  • Grouping modes — View by cavity (all runs), by run (all cavities), or cavity × run matrix with synced Y-axis for fair comparison.
  • Focus from triage — Click Focus on any box plot to filter the full six-pack to that cavity, run, or combination.
  • Fullscreen triage — Expand the triage grid for shop-floor review; Esc to exit.

Expert Help

  • Six-Pack expert review — Expanded condition guide for bimodal data, drift, zero-bounded GD&T, gauge resolution, and outliers with immediate and long-term actions.
  • SPC expert review — Dedicated common-cause vs special-cause narrative plus dashboard reading guide on the SPC tab.
  • Chart-level help — Pulsing insight bulbs and chart help modals for Xbar/R, capability, normality, and SPC sections with context-specific copy.
  • Capability & normality tooltips — Method blocks include short descriptions, applicability warnings, and links to deeper reference text.

Data & Export

  • Excel drawing export — Finds the metrology header row, preserves hidden drawing data for reload, and adds visible thumbnail previews in the workbook.
  • Project save/load — JSON project files retain adjustments, excluded outliers, dimension images, and settings.
  • Modular build — Source split into ES modules; npm run bundle produces the single-file Metrology Data Analyzer.html.
  • Release workflow — Version and changelog live in release/*.json; npm run release bumps the version and npm run bundle refreshes the start page.
  • Series filter fix — Standard analysis “Select All” can truly deselect every series; steel adjustment reloads per dimension.
